# RK3576 智能物联网控制盒定位方案

## 一、产品定位
本项目不再仅被定义为“海洋物联网监控平台”，而进一步明确为：

**RK3576 智能物联网控制盒**

它是一台部署在 RK3576 上的边缘控制中心设备，面向海洋场景，负责现场数据采集、协议转换、视频接入、本地录像存储以及向云平台推送数据。

---

## 二、核心能力

### 1. 数据采集
- 海洋传感器数据采集
- 串口 / Modbus / MQTT / TCP 等数据接入
- 现场设备在线状态感知

### 2. 协议转换
- 多协议统一接入
- 异构数据规范化
- 边缘规则流与转换逻辑处理

### 3. 视频接入
- GB28181 视频接入
- WVP 管理
- ZLMediaKit 流转发
- 视频监控与预览

### 4. 本地录像与存储
- 本地录像存储
- 数据缓存与断网续存
- 录像文件与索引管理

### 5. 云平台推送
- MQTT / HTTP / WebSocket 等方式上云
- 数据摘要推送
- 告警上云
- 视频状态 / 录像索引推送

---

## 三、业务逻辑闭环

### Step 1：现场设备接入
控制盒接入海洋传感器、视频点位、边缘设备和采集单元。

### Step 2：数据采集
控制盒通过多种协议采集现场数据。

### Step 3：协议转换
控制盒对异构协议进行解析、转换、统一封装。

### Step 4：边缘本地处理
在本地完成：
- 数据缓存
- 协议转发
- 视频接入
- 录像存储
- 状态聚合

### Step 5：云端推送
将关键数据、告警、状态与索引等推送到云平台。

---

## 四、前端结构建议

### 1. 登录页
定位：
- 控制盒本地入口页
- 极简
- 登录 / 注册
- 登录成功后直接进入指挥大屏

建议标题：
- `RK3576 智能物联网控制盒`

建议副标题：
- `数据采集 · 协议转换 · 视频接入 · 本地录像 · 云端推送`

### 2. 指挥大屏
定位：
- 控制盒主界面
- 边缘控制中心大屏

建议标题：
- `RK3576 边缘控制中心大屏`
- 或 `智能物联网控制盒指挥大屏`

显示内容建议：
- 控制盒总状态
- 数据采集概况
- 协议转换状态
- 视频接入概况
- 遥测趋势
- 视频矩阵
- 云端连接状态
- 当前登录用户
- 管理员后台入口

### 3. 后台管理
定位：
- 控制盒管理后台
- 仅管理员可进入

建议标题：
- `智能物联网控制盒后台`
- 或 `边缘控制中心后台`

---

## 五、后台菜单建议

建议菜单从“通用平台后台”重构为“控制盒能力后台”：

1. 控制盒总览
2. 用户管理
3. 数据采集
4. 协议转换
5. 视频接入
6. 录像存储
7. 云平台推送
8. 系统状态
9. 指挥大屏

---

## 六、界面内容重构方向

### 登录页文案
强调设备本体属性，而不是云平台属性。

### 指挥大屏文案
强调：
- 边缘控制中心
- 控制盒运行状态
- 本地采集与本地处理
- 视频与录像
- 云端连接

### 后台文案
强调：
- 控制盒配置与运维
- 本地能力管理
- 云推送配置
- 录像与存储管理

---

## 七、当前建议结论
后续所有页面不再仅以“海洋监控平台”描述自身，而统一升级为：

**海洋场景下的 RK3576 智能物联网控制盒 / 边缘控制中心**

这样更符合最终部署形态、产品形态和业务真实场景。
